---
title: Source Registry
description: Register custom search sources and build identifiers for providers.
---

Use Source Registry to add custom search providers and build valid Lavalink identifiers for your search queries.

## Register Custom Sources

Register new source aliases during app bootstrap.

```ts twoslash copy wrap title="register-sources.ts"
import { SourceRegistry } from 'hoshimi';

SourceRegistry.register({
  source: 'mysearch',
  name: 'my-provider',
  protocol: 'colon',
});

SourceRegistry.register(
  { source: 'mytts', protocol: 'double-slash' },
  { source: 'rawquery', protocol: 'raw' },
);

declare module 'hoshimi' {
  interface CustomizableSources {
    mysearch: 'my-provider';
    mytts: 'mytts';
  }
}
```

## Resolve & Validate

Check if a source is registered and retrieve its configuration.

```ts twoslash copy wrap title="resolve-sources.ts"
import { SourceRegistry } from 'hoshimi';

const source = SourceRegistry.resolve('my-provider');
const exists = SourceRegistry.isRegistered('mysearch');
const all = SourceRegistry.getRegistered();

console.log(source, exists, all);
```

## Build Identifiers

Create properly formatted identifiers for different protocol types.

```ts twoslash copy wrap title="build-identifiers.ts"
import { SourceRegistry } from 'hoshimi';

const colon = SourceRegistry.createIdentifier('mysearch', 'Daft Punk');
const doubleSlash = SourceRegistry.createIdentifier('mytts', 'hello world');
const raw = SourceRegistry.createIdentifier('rawquery', 'ytsearch:already-prefixed');

console.log(colon, doubleSlash, raw);
```

## Best Practices

- Register sources during application bootstrap.
- Use `resolve()` before searching if the source can be user-provided.
- Keep protocol choice consistent per source to maintain query format stability.
- Use module augmentation to add custom sources to TypeScript's `CustomizableSources` interface.
