---
title: Custom Structures & Types
description: Override built-in structures and use inferred structure types.
---

Hoshimi allows you to replace default classes with custom implementations. This guide covers both overriding structures and using their inferred types.

## Basic Override Pattern

Create a custom subclass and assign it to the `Structures` factory.

```ts twoslash copy wrap title="custom-player.ts"
import { Player, Structures } from 'hoshimi';

class CustomPlayer extends Player {
  public isDJModeEnabled(): boolean {
    return this.volume > 120;
  }
}

Structures.Player = (...args) => new CustomPlayer(...args);

interface CustomizableStructures {
  Player: CustomPlayer;
}
```

## Multiple Overrides

Override several structures at once for comprehensive customization.

```ts twoslash copy wrap title="multi-override.ts"
import { LyricsManager, Node, Player, Structures } from 'hoshimi';

class ExtendedPlayer extends Player {}
class ExtendedNode extends Node {}
class ExtendedLyricsManager extends LyricsManager {}

Structures.Player = (...args) => new ExtendedPlayer(...args);
Structures.Node = (...args) => new ExtendedNode(...args);
Structures.LyricsManager = (...args) => new ExtendedLyricsManager(...args);

interface CustomizableStructures {
  Player: ExtendedPlayer;
  Node: ExtendedNode;
  LyricsManager: ExtendedLyricsManager;
}
```

## Using Structure Types

The `*Structure` type aliases represent the final instance types. Use them in function signatures for flexibility.

```ts twoslash copy wrap title="structure-types-usage.ts"
import type { PlayerStructure, TrackStructure } from 'hoshimi';

function attachPlayerFeatures(player: PlayerStructure): void {
  // Works with default Player or any custom Player replacement
  console.log(`Guild: ${player.guildId}`);
}

function inspectTrack(track: TrackStructure): string {
  return track.info.title;
}
```

## Module Augmentation for Custom Data

Extend built-in structures with custom properties via module augmentation.

```ts twoslash copy wrap title="custom-data-types.ts"
import type { LyricsResult } from 'hoshimi';

declare module 'hoshimi' {
  interface CustomizablePlayerStorage {
    enabledAutoplay: boolean;
    enabledLyrics: boolean;
    lyricsId: string;
    lyrics: LyricsResult;
  }

  interface CustomizableTrack {
    requester: {
      id: string;
      username: string;
    };
  }
}
```

## Best Practices

- Override structures once during bootstrap (before creating any managers).
- Keep constructor signatures unchanged to maintain compatibility.
- Add narrow, domain-specific methods instead of rewriting core flow logic.
- Use `*Structure` types in function signatures to avoid coupling to concrete classes.
- Keep extensions strongly typed after overrides with `InferCustomStructure`.
