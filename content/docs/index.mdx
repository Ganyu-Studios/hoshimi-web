Hoshimi is a **TypeScript-first Lavalink v4 client** built with strict typing, clean
architecture, and extensibility in mind — easy to use and always up to date.

<Callout type="info" title="New to Lavalink?">
  Lavalink is a standalone audio-sending node. Hoshimi talks to it over REST and
  WebSocket and hands you a fully typed API, so you build music bots without
  touching the wire protocol yourself.
</Callout>

## Why Hoshimi?

<Cards>
  <Card title="Type-safe" description="Comprehensive typings and type guards catch mistakes before they ship — not at runtime." />
  <Card title="Modular" description="Nodes, players, queues and storage are separate concerns you can override safely." />
  <Card title="Batteries included" description="Autoplay (YouTube & Spotify), live lyrics, audio filters and session resumption out of the box." />
  <Card title="Scalable" description="Designed for bots running multiple nodes with resumable sessions and persistence." />
</Cards>

## Install

```bash
pnpm add hoshimi
```

Hoshimi runs on **Node.js 22+**, **Bun 1.3+**, or **Deno 2.5+** (unstable).

## First steps

Create the manager with your node list and a `sendPayload` function that forwards
voice updates through your Discord gateway connection.

```ts twoslash copy wrap title="src/index.ts"
const gatewaySend = async (guildId: string, payload: unknown): Promise<void> => {  };
// ---cut---
import { Hoshimi } from 'hoshimi';

const hoshimi = new Hoshimi({
  nodes: [
    {
      host: 'localhost',
      port: 2333,
      password: 'youshallnotpass',
    },
  ],
  sendPayload(guildId, payload) {
    return gatewaySend(guildId, payload);
  },
});
```
