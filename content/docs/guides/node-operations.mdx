## Select Nodes

Use `getLeastUsed()` for automatic load balancing, or access nodes directly.

```ts twoslash copy wrap title="node-selection.ts"
import type { Hoshimi } from 'hoshimi';

declare const manager: Hoshimi;

const leastUsed = manager.nodeManager.getLeastUsed();

if (!leastUsed) {
  throw new Error('No available nodes');
}

console.log(`Using node: ${leastUsed.id}`);
console.log(`Ready: ${leastUsed.ready}`);
console.log(`Penalties: ${leastUsed.penalties}`);
```

## Decode Tracks

Decode Lavalink track data to extract metadata.

```ts twoslash copy wrap title="track-decoding.ts"
import type { Node } from 'hoshimi';

declare const node: Node;

const single = await node.decode.single('QAAAyAIA...', 'user-id');
const many = await node.decode.multiple(['QAAAyAIA...', 'QAAByAIA...'], 'user-id');

console.log(single.info.title);
console.log(many.length);
```

## REST Operations

Access low-level REST endpoints for advanced use cases.

```ts twoslash copy wrap title="rest-operations.ts"
import type { Node } from 'hoshimi';

declare const node: Node;

const players = await node.rest.getPlayers();
console.log(`Active players: ${players.length}`);
```

## Best Practices

- Prefer `getLeastUsed()` for default load balancing.
- Always check `node.ready` before forcing node-specific operations.
- Use `node.rest` only for low-level operations; prefer manager/player abstractions for typical flows.
- Validate node health before critical operations.
