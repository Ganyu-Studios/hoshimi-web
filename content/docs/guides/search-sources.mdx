This guide covers how to search for tracks from the manager or player level, and how to handle different result types.

## Search from Manager

Use manager search to resolve tracks before creating or using a player.

```ts twoslash copy wrap title="manager-search.ts"
import { SearchSources } from 'hoshimi';
import type { Hoshimi } from 'hoshimi';

declare const manager: Hoshimi;

const result = await manager.search({
  query: 'Daft Punk - Harder Better Faster Stronger',
  source: SearchSources.Youtube,
  requester: {
    id: '123',
    username: 'demo-user',
  },
});

console.log(result.loadType);
console.log(result.tracks.length);
```

## Search from Player

Use player search in command handlers when the player already exists.

```ts twoslash copy wrap title="player-search.ts"
import { LoadType } from 'hoshimi';
import type { Player } from 'hoshimi';

declare const player: Player;

const result = await player.search({
  query: 'Lofi hip hop',
  requester: {
    id: '123',
    username: 'demo-user',
  },
});

if (result.loadType === LoadType.Track || result.loadType === LoadType.Search) {
  await player.queue.add(result.tracks[0]);

  if (!player.isPlaying()) {
    await player.play();
  }
}
```

## Handling Load Types

Always branch by `loadType` to handle different search outcomes:

- **`Track`**: Single track found.
- **`Playlist`**: Multiple tracks in a playlist.
- **`Search`**: Search results (multiple candidate tracks).
- **`Empty`**: No results found.
- **`Error`**: Search failed (provider error or network issue).

## Source Configuration

- Set a default source in manager options (`defaultSearchSource`) during bootstrap.
- Override per query with the `source` parameter in `SearchOptions` when needed.
- Build source identifiers using [Source Registry](/docs/guides/source-registry) for custom providers.
