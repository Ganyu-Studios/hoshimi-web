## Listening to Events

Subscribe to manager events using `EventNames` constants. Key events include node readiness, player creation, and queue completion.

```ts twoslash copy wrap title="event-listeners.ts"
import { Hoshimi as OriginalManager, HoshimiEvents } from 'hoshimi';

class Hoshimi extends OriginalManager {
  on<K extends keyof HoshimiEvents>(
    event: K,
    listener: (...args: HoshimiEvents[K]) => void
  ): this {
    return this;
  }
}

// ---cut---

import { EventNames } from 'hoshimi';

declare const manager: Hoshimi;

manager.on(EventNames.NodeReady, (node, retries) => {
  console.log(`Node ${node.id} ready (retry #${retries})`);
});

manager.on(EventNames.PlayerCreate, (player) => {
  console.log(`Player created for guild ${player.guildId}`);
});

manager.on(EventNames.QueueEnd, (player) => {
  console.log(`Queue ended for guild ${player.guildId}`);
});

manager.on(EventNames.PlayerDestroy, (player) => {
  console.log(`Player destroyed for guild ${player.guildId}`);
});
```

## Production Checklist

Ensure consistent state across your application:

- **Initialization**: Call `manager.init(botUser)` once after bot ready event.
- **Packet forwarding**: Forward all raw voice packets to `manager.updateVoiceState()` (see [Setup & Integration](/docs/getting-started/setup-and-integration#step-3-forward-voice-packets)).
- **Session persistence**: On `PlayerUpdate` events, save relevant player state (current track, position, loop mode, volume).
- **Cleanup**: On `QueueEnd` and `PlayerDestroy`, clear transient storage (message IDs, lyrics subscriptions, feature flags).
