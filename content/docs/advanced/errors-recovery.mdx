## Typed Error Handling

```ts twoslash copy wrap title="typed-errors.ts"
import {
  ManagerError,
  NodeError,
  OptionError,
  PlayerError,
  ResolveError,
  RestError,
  StorageError,
} from 'hoshimi';

try {
  // any operation that may fail
} catch (error) {
  if (error instanceof OptionError) {
    console.error('Invalid options:', error.message);
  } else if (error instanceof ManagerError) {
    console.error('Manager state error:', error.message);
  } else if (error instanceof NodeError) {
    console.error('Node connection error:', error.message);
  } else if (error instanceof PlayerError) {
    console.error('Playback error:', error.message);
  } else if (error instanceof ResolveError) {
    console.error('Track resolve error:', error.message);
  } else if (error instanceof RestError) {
    console.error('Lavalink REST error:', error.message);
  } else if (error instanceof StorageError) {
    console.error('Storage adapter error:', error.message);
  }
}
```

## Recovery Checklist

- Before commands: check manager health (`isUseable`).
- Before player operations: verify voice channel and player existence.
- For queue operations: guard with `queue.isEmpty()`.
- On reconnect-sensitive flows: persist player/session data after updates.
- On teardown (`QueueEnd`/`PlayerDestroy`): clear transient storage (lyrics/message IDs).
