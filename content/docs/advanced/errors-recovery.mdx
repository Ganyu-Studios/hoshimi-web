## Typed Error Handling

Use `instanceof` checks to handle different error categories appropriately.

```ts twoslash copy wrap title="error-handling.ts"
import {
  ManagerError,
  NodeError,
  OptionError,
  PlayerError,
  ResolveError,
  RestError,
  StorageError,
} from 'hoshimi';

try {
  // Any Hoshimi operation that may fail
} catch (error) {
  if (error instanceof OptionError) {
    console.error('Invalid options:', error.message);
  } else if (error instanceof ManagerError) {
    console.error('Manager state error:', error.message);
  } else if (error instanceof NodeError) {
    console.error('Node connection error:', error.message);
  } else if (error instanceof PlayerError) {
    console.error('Playback error:', error.message);
  } else if (error instanceof ResolveError) {
    console.error('Track resolution failed:', error.message);
  } else if (error instanceof RestError) {
    console.error('Lavalink REST error:', error.message);
  } else if (error instanceof StorageError) {
    console.error('Storage adapter error:', error.message);
  } else {
    console.error('Unknown error:', error);
  }
}
```

## Error Categories

| Error Type | Cause | Recovery |
|-----------|-------|----------|
| **OptionError** | Invalid configuration | Fix options, restart |
| **ManagerError** | Invalid manager state | Check manager initialization |
| **NodeError** | Node connection/status | Retry, fallback to another node |
| **PlayerError** | Invalid player operation | Validate player state |
| **ResolveError** | Track lookup failed | Try different search query |
| **RestError** | Lavalink HTTP error | Retry with backoff |
| **StorageError** | Adapter operation failed | Check storage backend |

## Recovery Checklist

Apply these patterns to ensure resilient playback:

- **Before commands**: Check `manager.isUseable()` for quick health validation.
- **Before player operations**: Verify voice channel access and player existence.
- **Queue operations**: Guard with `queue.isEmpty()` before shifting/accessing.
- **Session persistence**: Save player/queue state after successful updates.
- **Cleanup**: On `QueueEnd` and `PlayerDestroy`, clear transient storage (lyrics IDs, message refs).
